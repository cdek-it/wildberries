# Генерация токена Wildberries

Для генерации ключа, в [Личный кабинет партнера Wildberries](https://seller.wildberries.ru/) необходимо нажать
на название магазина и перейти в раздел `Настройки`.

<Image img={require('./img/main.png')} />

<details>
    <summary>Если на главной странице отображается предупреждение об отключении раздела `Маркетплейс`</summary>

    <Image img={require('./img/notice.png')} />

    Интеграция не сможет работать при отключенном разделе `Маркетплейс`. Необходимо выполнить предлагаемые действия
    для исправления настроек.

</details>

Со страницы профиля необходимо перейти в раздел `Доступ к API`.

<Image img={require('./img/profile.png')} />

Затем создать новый токен, нажав на кнопку **Создать новый токен**.

<Image img={require('./img/api.png')} />

<details>
    <summary>Если кнопка **Создать новый токен** недоступна</summary>

    <Image img={require('./img/unavailable.png')} />

    При недоступности функционал создания нового токена, необходимо проверить права доступа используемого аккаунта
    или обратиться в техническую поддержку Wildberries.

</details>

Для создаваемого токена необходимо указать название и выбрать права доступа. Затем
подтвердить создание нажатием на кнопку **Создать токен**.

<Image img={require('./img/create.png')} />

:::info

Для корректной работы, токен должен иметь сразу две категории доступа: `Контент` и `Маркетплейс` и отключенные
настройки `Тестовый контур` и `Только на чтение`.

:::

<details>
    <summary>Для чего нужны права доступа</summary>

    - Доступ к категории `Контент` необходим для получения информации о габаритах товаров для накладной.
    - Доступ к категории `Маркетплейс` необходим для получения информации о заказах.
    - Благодаря отключенному параметру `Тестовый контур` интеграция будет работать с реальными данными. Тестового
    режима для DBS API в настоящий момент не существует у Wildberries.
    - Благодаря отключенному параметру `Только на чтение` интеграция сможет изменять статус заказа при передаче его в доставку, а также вручению.

</details>

Необходимо скопировать сгенерированный токен и вставить его в [настройки интеграции](https://lk.cdek.ru/integration).

<Image img={require('./img/token.png')} />

:::warning

Токен будет доступен только один раз после создания. При его утере, необходимо будет создать новый.

:::

После заполнения поля, необходимо вернуться в [личный кабинет СДЭК](https://lk.cdek.ru), подтвердить ознакомление с
согласием на передачу данных и отправить токен на проверку, нажав **Проверить API-ключ**.

<Image img={require('./img/key.png')} />
